拖拽相关

拖拽操作 - Web API | MDN

前端拖拽API你会用了么-CSDN博客

拖拽主要就是先设置可拖拽元素，然后监听各个拖拽节点进行操作

注意：拖拽的时候，鼠标是一个禁止的符号：大多数网页和应用程序都不能放置拖拽数据，想要成功必须取消dragenter 和 dragover 事件的默认处理行为。 e.prevetDefault()解决

html
// 简单拖拽排序demo
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <style>
    .container {
      display: flex;
      flex-direction: column;
      justify-content: center;
      align-items: center;
    }

    .item {
      width: 400px;
      height: 50px;
      background-color: skyblue;
      margin-bottom: 20px;
      text-align: center;
      line-height: 50px;
      border-radius: 10px;
    }

    .move {
      background-color: pink;
    }
  </style>
  <body>
    <div class="container">
      <div class="item" draggable="true">1</div>
      <div class="item" draggable="true">2</div>
      <div class="item" draggable="true">3</div>
      <div class="item" draggable="true">4</div>
      <div class="item" draggable="true">5</div>
    </div>
    <script>
      let source; // 拖拽源

      const container = document.querySelector(".container");
      //开始拖拽
      container.addEventListener("dragstart", (e) => {
        // console.log("dragstart", e.target);
        source = e.target;
        setTimeout(() => {
          source.classList.add("move");
        }, 0);
      });
      // 拖拽进入元素
      container.addEventListener("dragenter", (e) => {
        e.preventDefault();
        if (e.target === source || e.target === container) return;
        const sourceIndex = Array.from(container.children).indexOf(source);
        const targetIndex = Array.from(container.children).indexOf(e.target);
        if (sourceIndex < targetIndex) {
          container.insertBefore(source, e.target.nextElementSibling);
        } else {
          container.insertBefore(source, e.target);
        }
      });
      // 拖拽经过元素
      container.addEventListener("dragover", (e) => {
        e.preventDefault();
        // console.log("dragover", e.target);
      });
      // 拖拽结束
      container.addEventListener("dragend", (e) => {
        source.classList.remove("move");
      });
    </script>
  </body>
</html>

2.动态路由实现方案

前置：Vite 文件系统 import.meta.glob

三、Vite 文件系统 import.meta.glob-CSDN博客

首先利用import.meta.glob获取对应正则匹配后的文件模块，然后使用公共合并方法进行合并

typescript
import type { RouteRecordRaw } from 'vue-router';

// 定义模块类型
interface RouteModuleType {
  default: RouteRecordRaw[];
}

/**
 * 合并动态路由模块的默认导出
 * @param routeModules 动态导入的路由模块对象
 * @returns 合并后的路由配置数组
 */
function mergeRouteModules(
  routeModules: Record<string, unknown>,
): RouteRecordRaw[] {
  const mergedRoutes: RouteRecordRaw[] = [];

  for (const routeModule of Object.values(routeModules)) {
    const moduleRoutes = (routeModule as RouteModuleType)?.default ?? [];
    mergedRoutes.push(...moduleRoutes);
  }

  return mergedRoutes;
}

export { mergeRouteModules };

export type { RouteModuleType };

const dynamicRouteFiles = import.meta.glob('./modules/**/*.ts', {
  eager: true,

const accessRoutes = [...dynamicRoutes, ...staticRoutes];

异步组件

  1. 懒加载：
    • 普通引入组件：组件会在应用启动时立即加载，无论它是否立即需要。这可能会导致初始加载时间变长，特别是当应用有很多组件或组件很大时。
    • 异步组件：组件会在需要时才加载，即当组件首次被渲染到页面上时才会加载。这样可以减少初始加载时间，提高应用的性能。
  2. 代码分割：
    • 普通引入组件：所有组件代码都会被打包到一个或几个大的 JavaScript 文件中，导致文件体积较大。
    • 异步组件：每个异步组件会被单独打包成一个小的 JavaScript 文件，只有在需要时才会加载这个文件。这有助于实现代码分割，减小初始加载的文件体积。
  3. 加载状态处理：
    • 普通引入组件：没有内置的加载状态处理机制。
    • 异步组件：defineAsyncComponent 提供了加载状态的处理机制，可以自定义加载中、加载失败等状态。例如，可以显示一个加载指示器或错误信息。
  4. 高级配置：
    • 普通引入组件：没有提供高级配置选项。
    • 异步组件：defineAsyncComponent 允许你传递一个配置对象，可以指定加载中组件、错误组件、加载超时时间等。例如：
js
const FormDialog = defineAsyncComponent({
  loader: () => import('./form.vue'),
  loadingComponent: LoadingComponent, // 加载中显示的组件
  errorComponent: ErrorComponent, // 加载失败显示的组件
  delay: 200, // 延迟显示加载中组件的时间（毫秒）
  timeout: 3000 // 加载超时时间（毫秒）
});

‌地图坐标标准‌是指用于确定地球上某一位置的具体坐标系统。不同的地图服务或应用可能会采用不同的坐标标准，以满足不同的需求和规范。以下是几种常见的地图坐标标准及其特点：

  1. ‌ WGS-84 ‌：这是全球通用的地心坐标系，广泛用于 GPS 设备和其他全球定位系统。WGS-84坐标系由美国国防部制定，用于确定地球表面的精确位置‌12。
  2. ‌ **GCJ-02 （火星坐标系）**‌：这是中国国家测绘局发布的坐标系，用于国内地图的加密处理。所有在中国境内的互联网地图必须至少使用GCJ-02坐标系进行首次加密。高德、 百度 和 腾讯 等地图服务在中国境内均使用此坐标系‌。
  3. ‌ **BD-09 （百度坐标系）**‌：这是百度在GCJ-02基础上进行的二次加密，具有更高的安全性和隐私保护。百度地图使用此坐标系来确保位置数据的准确性和安全性‌。
  4. ‌ CGCS2000 ‌：这是中国的国家大地坐标系，与WGS-84非常接近，主要用于国家测绘和地理信息系统应用。CGCS2000的定义与WGS-84实质相同，但在某些应用中可能会存在微小差异‌

坐标转换方法

不同坐标系之间的转换通常需要特定的算法。例如，WGS-84坐标系可以转换为GCJ-02坐标系，而百度地图的BD-09坐标系则是在GCJ-02基础上进一步加密。这些转换过程通常由专业的地图服务提供商完成，以确保数据的准确性和一致性‌

git fetch --prune origin

git fetch origin

  • 从远程仓库（默认名为 origin）下载最新的提交、分支和标签信息到本地，但不会自动合并或修改你的本地分支。
  • 这相当于同步远程仓库的最新状态到本地（比如其他人新推送的分支）。

--prune 的作用

  • 清理本地无效的远程分支引用：如果远程仓库的分支已被删除（例如其他人执行了 git push origin --delete branch-name），这个选项会同步删除本地存储的对应远程分支引用（即 origin/branch-name 这种形式的引用）。
  • 如果不加 --prune，本地会残留这些已删除远程分支的引用，导致 git branch -a 显示一堆过时的分支。
